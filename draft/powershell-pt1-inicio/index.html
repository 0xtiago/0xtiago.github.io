Introdução e motivações

Este é o primeiro post de uma série que se inicia, servindo de documentação durante meus estudos para me aperfeiçoar em ataques e métodos de defesa Windows Active Directory.

Durante minha carreira profissional, sempre fui um entusiasta do software livre, focando sempre em soluções que pudessem ser customizadas e executadas com estabilidade em sistemas operacionais GNU/Linux, conforme o tempo foi passando fui cada vez mais enfiando minha cabeça num mundo paralelo chamado Shell Script. Não quero dizer que negligenciei meus estudos em ambientes Windows, tanto é que trabalhei por muitos anos administrando ambientes híbridos, como por exemplo ambientes baseados em Windows (Active Directories, Desktops, DNS, DHCP, File Servers, ERPs..) junto com Linux (Proxy, Firewall, IDS, SIEM…), mas meu foco era sempre Linux, fazendo o mínimo necessário em ambientes Windows apenas focando em facilidade, estabilidade e segurança.

Por quê você deve aprender PowerShell?

Se você fosse o responsável em contratar todos os administradores de sistema, quem você preferiria ter em sua equipe? Uma pessoa que executa manualmente suas tarefas diárias usando as ferramentas básicas. Ou um administrador que leva segundos para executar as mesmas tarefas de maneira automatizada?

Explorando comandos do PowerShell

Como toda linguagem, o PowerShell tem comandos, um nome genérico para nomes de expressões executáveis. Um comando pode ser qualquer coisa, desde um ipconfig a um Get-Content, e obviamente você também pode criar os seus próprios comandos. Mas se você digitar algum comando não existente, você obterá uma mensagem como a de baixo.

PS C:\Users\tiago> nada

was included, verify that the path is correct and try again.

PS C:\Users\tiago>

Você pode executar Get-Command para ver a lista de todos os comandos disponíveis por padrão no PowerShell. Você perceberá um padrão Verbo-Substantivo nos comandos, assim sendo mais intuitivo para os usuários. Apesar de ser opcional a adoção deste padrão nos seus próprios comandos, é extremamente recomendável que siga este padrão.

O PoweShell possui tipos de comandos: cmdlets, funções, aliases e as vezes scripts externos. A maioria dos comandos nativos da Microsoft são cmdlets, que são comandos tipicamente escritos em outras linguagens como C#. Ao executar o comando Get-Command você poderá observar que tipo de comando é.

PS C:\Users\tiago> Get-Command -Name Get-Alias

CommandType     Name                                               Version    Source
-----------     ----                                               -------    ------
Cmdlet          Get-Alias                                          3.1.0.0    Microsoft.PowerShell.Utility

Funções são comandos escritos em PowerShell. Você pode escrever funções para realizar tarefas que precisar. Cmdlets e funções são os tipos de comandos mais comuns no PowerShell.

Com o comando Get-Command você poderá explorar todo o universo funções e cmdlets do PowerShell. Quando utilizado sem qualquer parâmetro, demorará um pouco para terminar a listagem de comandos.

Percebeu algo diferente além de Get? Algo como Set, Update e Remove? Os comandos são nomeados para ajudar o usuário!

Execute o PowerShell como Administrador, pois possivelmente irá alterar alguns registros.

PS C:\Users\tiago> Update-Help

Conceitos

Variáveis

Visualizando e alterando variáveis

Todas as variaveis iniciam com um cifrão ($), assim o PowerShell entende que você quer obter o valor de uma variavel, e não tentando executar um comando.

PS> $MaximumHistoryCount
4096

Você também pode alterar uma variável.

PS> $MaximumHistoryCount = 200
PS> $MaximumHistoryCount
200

Criando variáveis

Assim como qualquer linguagem podemos criar variaveis.

PS> $cor = 'vermelho'
PS> $cor
vermelho

No exemplo anterior utilizamos a forma mais fácil de se criar e declarar uma variável. Poderíamos fazer de outra forma.

PS> Set-Variable -Name cor -Value vermelho
PS> Get-Variable -Name cor

Name                           Value
----                           -----
cor                            vermelho

PS> $foo = 1
PS> $foo = 'um'

Execution Policy

O Execution Policy não é uma medida de segurança, mas sim uma forma de prevenir o usuário de acidentalmente excutar scripts por acidente.

powershell -ExecutionPolicy bypass
powershell -c cmd
powershell -endodedcommand $env:PSExecutionPolicyPreference="bypass"
